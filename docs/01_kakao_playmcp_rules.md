## 3. ⭐ 서버 개발가이드 (반려 방지 필수)

### 3-1. 서버 생성 조건
- MCP 버전 **`2025-03-26` ~ `2025-11-25`**
- **Streamable HTTP** 만 지원 / **Remote MCP** 만 (공개 URL) / **Stateless 권장**
- 인증 필요 시 OAuth 또는 커스텀 헤더 (※ 본 서비스는 **인증 불필요 설계 지향**)
- **MCP Inspector** 사전 점검 / 공식 SDK 사용
- ❌ 서버명·툴명에 **`kakao`** 금지 (대소문자·위치 불문)

### 3-2. Tool 구성
- 이름: 1~128자, `A-Z a-z 0-9 _ -` 만, 중복 금지, **대소문자 구분**
- 개수: 최대 20 (초과 금지), **3~10개 권장**
- 필수 property: `name`, `description`, `inputSchema`, `annotations`
- **annotations 5종 모두 지정**: `title`, `readOnlyHint`, `destructiveHint`, `openWorldHint`, `idempotentHint`
- description: **영문 권장**, **1,024자 이내**, **서비스명 포함**
  - 예: `... from Saju Concierge(사주 컨시어지)`

### 3-3. 응답 형식
- PlayMCP AI채팅: **`TextContent` 타입만** 허용 (도구함→Claude/ChatGPT 직접 연결 시 타입 확장 가능)
- **Markdown 권장**, API JSON 원문 금지(정제)
- 이미지: `![설명](https://.../img.png)`
- **Response 24k 초과 → 반려.** result 최소화
- ※ PlayMCP는 Resource·Prompt 미지원 → **Tool 중심 설계**

### 3-4. 운영(성능)
- **평균 100ms / p99 3,000ms 필수** (명식 계산은 메모리 연산이라 유리)
- ❌ 광고 노출 유도 금지

## 4. MCP 심사 정책 (반려 사유)

- Tool ≥1개 (권장 3~20)
- 표준 스펙 위배·비정상 동작 → 반려
- ❌ Zapier 등 자동생성 MCP 불가
- ❌ 명칭·출력만 바꾼 동일구조 MCP 반복 불가
- ❌ **"LLM 웹검색으로 충분히 구현 가능한 기능"만 → 반려/공개제한** (확장·추가 목적 명확하면 예외)
  - → 본 서비스는 **자체 만세력 계산 + 자체 해석 데이터**로 고유가치 확보 (단순 LLM 운세 아님)
- 추상적·모호한 이름/설명 → 수정 요청
- ❌ 도구 실행에 유료 결제/구독 필수 → 반려
- 느림·timeout·과도한 외부호출 → 반려
- 아웃링크 OK, **상업 링크·구매유도·리워드 과다 → 반려**
- 비속어/정치/성적/사회통념 위배 → 반려 (운세 = 엔터테인먼트 톤 유지)
- **개인정보 6종 수집·전송 → 반려**: 주민등록번호/운전면허번호/여권번호/외국인등록번호/카드번호/계좌번호
  - ※ 생년월일·태어난 시각은 6종에 해당 안 됨. 단 **저장하지 않고 계산 후 폐기**.
- 대표 이미지: 움직이는 이미지 불가, 서비스 성격 부합·고품질

## 5. 배포 (PlayMCP in KC, `https://playmcp.kakaocloud.io`)
- 계정당 **최대 2대**. (Concierge가 1대 사용 중·Active, Saju가 2대째)
- **방법 A — Git 소스 빌드(Saju 채택)**: 서버명/설명/**Git URL(루트 Dockerfile 필수)**/브랜치/Dockerfile 경로/PAT(private만)
- **방법 B — 컨테이너 이미지**: **`linux/amd64` 필수**(arm64 실패)
- 삭제 복구 불가.
- 🛑 **실전 함정(첫 프로젝트 확인): KC에는 환경변수(API 키) 입력란이 없다.** 두 방식 모두. 키가 필요하면 빌드 시점에 이미지로 주입해야 함(GitHub Secrets→비공개 ghcr). **단 Saju는 외부 키가 0개라 이 문제를 겪지 않는다 → Git 소스 빌드(public)로 깔끔 배포.** 상세 `docs/09`.
- **Dockerfile은 `--platform=linux/amd64` 명시.** 헬스체크 `GET /`로 배포 직후 상태 확인.

## 6. 등록·심사·공개 절차
임시 등록 → "정보 불러오기" 성공 확인 → 도구함 추가 → AI채팅 테스트 → **대화 예시 3개 입력** → 심사 요청 → 승인 → **"전체 공개" 전환** → 상세 URL 복사 → 공모전 비즈폼 "Player 예선 참여"(최대 2개, **1회 제출**).
- ⚠️ **"정보 불러오기"가 성공해야 함**(실패=엔드포인트/툴스펙 문제). 그다음 **"임시 등록"**(절대 "등록 및 심사요청" 누르지 말 것) → "MCP 상세 미리보기"→"도구함에 추가"→AI채팅 테스트.
- **Claude 커넥터로도 테스트 가능**: PlayMCP가 Claude 공식 커넥터로 등록돼 있어, Claude Pro/MAX 설정→커넥터→PlayMCP 연결(카카오 OAuth) 후 도구함 도구를 Claude에서 직접 호출(시연·본선 데모 유용).
- 반려 시 카카오계정 대표이메일로 사유 발송 → 임시저장 상태 → 수정 후 재심사 요청.
- 상세 절차·실전 함정은 `docs/09_kc_deploy_playbook.md`.
- ⚠️ **등록 후 툴 변경 시 "정보 불러오기" 재실행 → 재심사 유발**(노션 원문 확인). 심사요청 전 툴셋 확정. **AI채팅 동시 적용 MCP 최대 10개.** 대화예시 최대 3개. 대표이미지 움짤 금지.

## 7. 응모 직전 체크리스트 (= /check)
