# 02. 제품 정의 + 통합 툴 스펙 (Saju Concierge)

## 1. 한 줄 정의
생년월일시를 입력하면 사주 명식을 계산해 **오늘의 기운·올해 운세·궁합·이름풀이**를 재미있게 풀어주고, **버튼/선택지를 눌러 이어가는** 한국형 운세 MCP.

## 2. 타깃 & 가치
- **타깃**: 운세를 보는 거의 모든 한국인 (점심 다음으로 자주 찾는 게 "오늘 운세")
- **대중성**: 운세 시장 규모가 크고, 매일 보는 중독성 + 궁합 같은 공유성 ↑. 카카오톡에서 친구 궁합 보기는 자연스러움.
- **시의성/차별성**: 해외 소비자 MCP는 점성술뿐. 국내 사주 MCP는 **이미 존재**(1FATE '정통 명리학', '사주봄')하나 모두 **정통·전문가형**(격국·용신·신살). 우리는 **재미·공유·매일 습관·대중 접근성**(사주 MBTI 유형·데일리 럭키 키트·공유 궁합 카드·칩 여정·카톡 데일리 발송)으로 **명확히 다른 세그먼트**. ⚠️ 심사정책: "명칭·문구·출력형식만 바꾼 MCP는 신규 불인정(클론 반려)" → 시그니처로 차별화가 생존조건.
- **공유·바이럴(D-114)**: 모든 결과에 '공유 카드'(귀여운 UI) + '나도 보기' 수신자 CTA + 서비스명 → 공유받은 사람이 바로 우리 서비스를 따라 쓰게 유도(공개투표 모수 확장). `src/lib/share.ts`.
- **포지셔닝**: "사주를 풀어주는 분석가"가 아니라 **"사주로 오늘을 운영하는 데일리 비서"**(처방=럭키 색/음식/방향/시간대·do&don'ts, 매일 카톡 발송, 공유 궁합).

## 3. 왜 반려 안 되나 (고유가치)
- 실시간 운세는 LLM이 못 줌. **만세력 계산(명식)**은 실제 알고리즘이고, **십신·오행 해석 데이터**는 우리가 직접 구조화 → "LLM이 그냥 답하는 것"과 명확히 구분.
- 계산의 정확성은 manseryeok-js(KASI 데이터)가 보장, 해석의 매력은 우리 데이터 + LLM 표현.

## 4. 톤 & 안전 (필수)
- **엔터테인먼트 톤 고정**: "재미로 보는 오늘의 기운". 단정적 예언·공포 조장 금지.
- **면책 문구** 짧게 포함: 재미·참고용임을 명시.
- 건강·죽음·중대사 단정 금지. 부정적 운세도 **건설적·완곡하게**.
- 생년월일시는 **계산 후 폐기**(stateless), 저장·외부전송 없음.

## 5. 통합 툴 목록 (7개 — 3~10 권장 충족)

| # | 툴명 (영문, kakao 금지) | 역할 |
|---|---|---|
| 1 | `computeSajuChart` | 생년월일시 → 명식(4기둥)+오행분포 계산·표시 (기반 툴) |
| 2 | `getTodayFortune` | 오늘의 기운/운세 (일진 대비 명식) |
| 3 | `getYearlyFortune` | 올해/특정 연도 운세 흐름 (세운·대운 개략) |
| 4 | `getCompatibility` | 두 사람 궁합 (명식 2개 비교) |

## 6. 입력 최소화 원칙 (UX 핵심)
- 첫 호출에서 생년월일시·성별·양력/음력을 받되, 이후엔 **버튼으로 이어가기**(다시 입력 X).
- stateless라 명식을 들고 다닐 수 없으므로, 응답에 **짧은 명식 요약 코드**를 포함하고 후속 툴이 그것을 재사용하도록 설계 (`04`, `03` 참조).
- 시간 모를 때 **"시 모름" 모드**(3주 계산) 지원 → 진입장벽↓.

## 7. 비범위 (하지 않는 것)
- 자미두수·서양 점성술 (1차 범위 밖, 추후 검토) — 단 AGPL 코드 사용 금지 원칙 유지
- 단정적 예언/운명론, 유료 결제 게이팅, 푸시 알림(MCP 구조상 불가)
- 타로 **카드 이미지** 사용 시 저작권 확인 전까지 텍스트 의미만 (docs/05)
