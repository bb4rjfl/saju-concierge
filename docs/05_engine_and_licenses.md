## 1. 계산 엔진 — manseryeok-js (확정)

- 패키지: **`@fullstackfamily/manseryeok`** (npm)
- 저장소: github.com/urstory/manseryeok-js
- **라이선스: MIT** ✅ (상업적 이용·수정·재배포 자유, 저작권 고지만 유지)
- 근거: **KASI(한국천문연구원)** 음양력 데이터 기반, DB 불필요 순수 JS/TS, 활발한 유지보수(~2026.3), 진태양시 보정·절기 기준 월주 등 정확도 검증.
- **확인된 API (v1.0.8, 2026-06-27 검증)**:
  - `calculateSaju(solarY, solarM, solarD, hour?, minute?, {longitude=127, applyTimeCorrection=true})` → `SajuResult` { yearPillar/monthPillar/dayPillar (한글 간지 예 "경오"), *Hanja, **hourPillar: string|null**, gapja, isTimeCorrected, correctedTime? }.
  - **시 모름 = hour 미입력 → `hourPillar: null`** (시 모름 모드 네이티브 지원).
  - **일간 = `dayPillar[0]`**(일주 천간). 천간/지지 = 간지 문자열 분해.
  - **입춘 년주 경계·절기 기준 월주(v1.0.7)를 엔진이 내부 처리** → 우리는 신뢰.
  - 음력: `lunarToSolar(ly, lm, ld, isLeapMonth?)` → `.solar`로 변환 후 calculateSaju.
  - 범위검증: `isSupportedYear(y)`(1900~2050) / `getSupportedRange()`. 예외 `OutOfRangeError`/`InvalidDateError`.
  - `exports` 맵: `import`→`.mjs`(ESM), `require`→`.js`(CJS), `types`→`.d.ts` → 우리 ESM/NodeNext에서 깔끔 해석.
  - 우리 래퍼: `src/engine/chart.ts`(computeChart) — 4기둥 + 오행 분포 + 십신 + 띠. 자체 명리 데이터는 `src/engine/elements.ts`.
- stateless·메모리 연산 → p99 3s 제약에 유리(외부호출 0; 날씨는 선택적 1개).

### 대안 (보류)
- `sajupy` (Python, MIT) — Python 런타임 갈 경우만. 성숙도 낮음(릴리스 없음).

## 2. 🛑 사용 금지 (라이선스)

- **orrery / @orrery/core** — **AGPL-3.0**. 코드 복붙·수정·의존 **절대 금지.**
- **Swiss Ephemeris** — **AGPL 또는 유료 상용 듀얼.** 점성술 계산 코드 사용 금지.
- 그 외 (A)GPL/SSPL/상용 제한 라이선스 전부 금지.

### 왜 치명적인가
AGPL은 **네트워크 서비스로 제공만 해도** 파생물 전체의 소스 공개를 강제한다. 우리는 MCP 서버를 KC에 올려 서비스하므로, AGPL 코드를 한 줄이라도 섞으면 **우리 서버 전체가 AGPL로 전염**되어 비공개·상업화 경로가 막힌다.

### 허용되는 참조 (코드 아님)
- AGPL 프로젝트라도 **명리학 개념·공식**은 참고 가능(공식·알고리즘 아이디어엔 저작권 없음). 예: "월주는 절기 기준", "시주는 진태양시 보정", "십신 = 일간 대비 오행 상생상극".
- 단 **코드·주석·데이터 파일을 복사하지 않는다.** 개념을 이해한 뒤 **우리 코드로 새로 작성.**

## 3. 해석 데이터 — 자체 작성 (차별화 + 반려 회피)

해석을 통째로 주는 MIT 라이브러리는 없다. 대신 **공개된 명리학 상식을 우리 말로 직접 구조화**해 데이터로 만든다. 이게 "LLM이 그냥 답하는 것"과 다른 고유가치다.

### 만들 데이터 (모두 자체 작성, 출처 문장 복사 금지)
- **천간 10 / 지지 12**: 오행·음양 매핑 (일부 manseryeok-js 제공값 활용 가능)
- **오행 5 (목화토금수)**: 성격 키워드 + 상생/상극 관계표
- **십신 10**: 비견·겁재·식신·상관·편재·정재·편관·정관·편인·정인 → 의미·강약 키워드·어울리는 분야
- **십이운성 12**: 장생~절 단계별 의미
- **주요 신살 소수**: 도화·역마·화개 등 대중적인 것 위주(과하면 복잡·불안 조장)
- **오행 균형 판정 규칙**: 과다/부족 → 해석 가이드
- **(선택) 타로 78장**: 정/역 의미 텍스트 (라이더-웨이트 전통 의미를 우리 말로)

### 데이터 작성 원칙
- **우리 문장으로.** 특정 사이트(블로그/위키 등) 텍스트 복붙 금지(저작권). 개념만 취해 재작성.
- 데이터 파일은 코드 리포에 포함(JSON/TS), 출처는 "공개된 일반 명리학 지식"으로 표기.
- 톤: 엔터테인먼트·건설적. 단정·공포 금지.

## 4. 해석 파이프라인 (3층)

```
[1] manseryeok-js → 명식(4기둥) + 오행분포 + 일간   (정확한 계산)
        ↓
[2] 자체 규칙 데이터 → 십신 매핑 / 오행 균형 / 십이운성 / 신살   (구조화된 해석 요소)
        ↓
[3] LLM 표현 → 위 요소를 자연스러운 한국어 운세 문장으로   (매력적인 표현)
```
- [1][2]는 우리 코드가 결정론적으로 처리(정확성·재현성).
- [3]은 MCP 응답을 받은 LLM이 담당. 서버는 [2]까지의 **구조화 결과 + 톤 가이드**를 Markdown으로 제공.
- 핵심: **계산·분류는 우리 것, 표현은 LLM.** → 반려(=LLM 단독 가능) 회피.
